# 1. UI 设计规范
----------------------------------------
## 1.1 Windows 版本（Electron/Qt）
- 100% 仿照 QQ UI（主界面、列表、聊天窗口）
- 风格必须符合苹果设计语言：  
  - 图标优先于文字  
  - 简洁、统一配色  
  - 平滑动画、布局一致性

## 1.2 Android 版本
- 不许使用虚幻引擎
- 必须使用开源 UI 框架（Qt/Flutter/Compose 等）
- 同样遵循 QQ 风格 + 苹果设计语言

----------------------------------------

# 3. 需要 Codex 提供（必须）的接口能力
----------------------------------------
（对应 bdtd.txt）
- 登录接口  
- KCP 连接状态接口  
- 消息发送 / 接收  
- 群聊消息 API  
- 加密基础类型（EncInt / EncString / EncJson 等）  
- 本地加密聊天记录读写接口  
- 文件加密与上传接口  

----------------------------------------
# 4. 需要 Codex 输出的文档
----------------------------------------
- bdtd.txt（完整 API 文档）
- EncryptedBasicTypes.md（加密基础类型说明书）
- 网络协议说明（握手过程、消息格式）
- 错误码说明

----------------------------------------
# 5. 前端数据结构约束
----------------------------------------
前端不得使用普通 string / int 等类型与后端交互。

必须使用 Codex 提供的：
- EncInt  
- EncLong  
- EncString  
- EncJson  

以防止被 IDA 分析。

----------------------------------------
# 6. 调用 DLL/SO 的方式
----------------------------------------
- 所有调用必须通过统一的前端封装层  
- 不得在 UI 组件中直接调用后端 API  
- DLL/SO 初始化必须在 UI 渲染前完成  
- 所有接口调用必须异步封装（Promise / Future）

----------------------------------------
# 7. 异常与日志要求
----------------------------------------
- 前端需要一个统一日志收集模块（不记录敏感信息）  
- 无论何时，崩溃日志必须写入本地沙盒目录  

----------------------------------------
# 8. 前端禁止做的事情
----------------------------------------
- 禁止直接接触任何密钥数据  
- 禁止访问真实服务器（使用假数据直到 Codex 完成协议）  
- 禁止存储明文消息、明文文件  
- 禁止修改 DLL/SO 内容
